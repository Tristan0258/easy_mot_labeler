# 多类别 MOT 数据标注工具项目需求文档（Codex 开发版）

## 1. 项目概述

### 1.1 项目名称

多类别 MOT 数据标注工具。

建议工程名：`EasyMOT-Labeler`。

### 1.2 项目定位

本项目是一款基于 Python + PyQt/PySide 的桌面端多类别多目标跟踪数据标注工具，用于视频或图像序列中的目标框标注、类别管理、Track ID 管理、自动保存、自动跟踪辅助、质量检查和 MOT 数据导出。

工具应强调：

1. **傻瓜式操作**：通过向导完成项目创建、数据导入、类别配置、标注、检查和导出。
2. **界面友好**：布局清晰，按钮直观，提示明确，默认深色主题。
3. **高效标注**：支持快捷键、复制上一帧、区间插值、自动跟踪辅助。
4. **自动保存**：标注过程中自动保存，支持异常恢复。
5. **格式兼容**：第一阶段重点支持内部 JSON 和 MOTChallenge 格式。
6. **适合 Codex 开发**：需求拆分明确，便于按模块逐步实现。

### 1.3 参考改进方向

本工具可参考 DarkLabel 的轻量化标注流程，但需要重点优化：

- 新手引导更清晰。
- 界面更现代。
- 操作更简单。
- 自动保存更可靠。
- Track ID 管理更直观。
- 自动跟踪辅助更易用。
- 导出和质量检查更规范。

### 1.4 第一阶段目标

第一阶段目标是完成一款可独立运行的桌面端标注工具，实现从导入视频到导出 MOT 标注结果的完整闭环。

第一阶段必须具备：

- 新建项目向导。
- 视频或图像序列导入。
- bbox 标注与编辑。
- 类别管理。
- Track ID 分配与查找。
- 帧浏览与播放。
- 基础快捷键。
- 自动保存与恢复。
- 复制上一帧标注。
- 区间线性插值。
- 基于 OpenCV 的自动跟踪辅助。
- 基础质量检查。
- MOT 格式导入导出。
- 深色主题。

## 2. 总体设计原则

### 2.1 傻瓜式操作原则

工具应让用户通过简单流程完成主要任务：

```text
启动软件
  → 新建项目
  → 选择视频或图像序列
  → 配置类别
  → 开始标注
  → 自动保存
  → 运行质量检查
  → 导出 MOT 数据
```

用户第一次打开软件时，应能在 3 分钟内完成一次简单标注和导出。

### 2.2 默认配置优先原则

软件应提供合理默认值，避免新用户被复杂配置阻塞。

默认设置：

- 默认启用自动保存。
- 默认启用深色主题。
- 默认 bbox 显示 `类别 + Track ID`。
- 默认 MOT 导出帧号为 1-based。
- 默认快捷键对齐常用软件。
- 默认自动创建项目目录。
- 默认自动生成内部标注文件。
- 默认启用导出前质量检查。

### 2.3 可见即可操作原则

主界面中必须直接展示常用按钮：

- 新建项目
- 打开项目
- 保存
- 上一帧
- 下一帧
- 播放 / 暂停
- 复制上一帧
- 插值
- 自动跟踪
- 质量检查
- 导出

### 2.4 安全操作原则

高风险操作必须支持撤销或确认。

高风险操作包括：

- 删除多个标注框。
- 批量修改类别。
- 批量修改 Track ID。
- 覆盖已有标注。
- 覆盖导出文件。
- 清空当前帧标注。

### 2.5 简洁优先原则

第一阶段不追求复杂功能堆叠，优先完成稳定、易用、可导出的基础闭环。

暂不实现：

- Track ID 合并。
- Track ID 拆分。
- 复杂快捷键自定义界面。
- 深度学习 AI 预标注。
- 多人协作。
- Web 端。

## 3. 用户角色与典型场景

### 3.1 新手标注员

需求：打开视频即可画框、选类别、分配 ID、自动保存。

系统应提供：

- 新建项目向导。
- 默认类别配置。
- 明确的按钮和提示。
- 错误操作可撤销。
- 清晰的保存状态。

### 3.2 熟练标注员

需求：通过快捷键、复制、插值和自动跟踪快速标注连续帧。

系统应提供：

- 基础快捷键。
- 快速帧导航。
- 复制上一帧。
- 区间插值。
- 自动跟踪当前目标。

### 3.3 审核员

需求：快速定位异常标注。

系统应提供：

- 质量检查面板。
- 异常帧跳转。
- bbox 越界检查。
- 重复 ID 检查。
- 未确认自动生成框检查。

### 3.4 算法工程师

需求：获得可用于训练或评估的 MOT 格式数据。

系统应提供：

- MOT 格式导出。
- `seqinfo.ini` 生成。
- 导出前校验。
- 内部 JSON 标注数据。

## 4. 软件主流程需求

### 4.1 启动页

软件启动后显示启动页，包含：

- 新建项目。
- 打开项目。
- 最近项目列表。
- 帮助入口。
- 当前软件版本。

### 4.2 新建项目向导

点击“新建项目”后打开向导窗口。

#### 第 1 步：选择数据源

支持选择：

- 视频文件。
- 图像序列文件夹。

支持格式：

- 视频：mp4、avi、mov、mkv。
- 图像：jpg、jpeg、png、bmp。

#### 第 2 步：设置项目名称和保存位置

系统默认使用视频文件名或图像文件夹名作为项目名。

系统自动创建项目目录。

#### 第 3 步：设置类别

用户可以：

- 使用默认类别。
- 手动添加类别。
- 从 YAML / JSON 导入类别。

默认类别示例：

```yaml
classes:
  - id: 1
    name: person
    color: "#ff4d4f"
    shortcut: "1"
  - id: 2
    name: vehicle
    color: "#52c41a"
    shortcut: "2"
  - id: 3
    name: bicycle
    color: "#1890ff"
    shortcut: "3"
```

#### 第 4 步：确认并创建

系统展示：

- 数据源路径。
- 项目保存路径。
- 视频帧数或图像数量。
- 分辨率。
- FPS。
- 类别数量。

点击“开始标注”后进入主界面。

### 4.3 主标注流程

```text
打开项目
  → 显示第 1 帧
  → 用户绘制 bbox
  → 选择类别
  → 分配 Track ID
  → 切换下一帧
  → 复制上一帧或自动跟踪
  → 修正 bbox
  → 自动保存
  → 质量检查
  → 导出 MOT
```

### 4.4 导出流程

点击“导出”按钮后打开导出向导：

1. 选择导出格式。
2. 选择导出路径。
3. 运行导出前检查。
4. 显示检查结果。
5. 确认导出。
6. 显示导出完成结果。

默认导出格式为 MOTChallenge。

## 6. 标注功能需求

### 6.1 bbox 绘制

用户在画布上按住鼠标左键拖拽生成 bbox。

绘制完成后：

1. 如果已选择默认类别，则自动使用默认类别。
2. 如果未选择类别，则弹出类别选择框。
3. Track ID 默认使用下一个可用 ID。
4. 用户可在右侧属性面板修改类别和 ID。

### 6.2 bbox 选择

支持：

- 单击选择单个 bbox。
- Shift + 单击多选。
- Esc 取消选择。

### 6.3 bbox 编辑

支持：

- 鼠标拖拽移动。
- 鼠标拖拽边缘或角点调整尺寸。
- 方向键移动 1 像素。
- Shift + 方向键移动 10 像素。

### 6.4 bbox 删除

按 Delete 或 Backspace 删除选中 bbox。

若选中数量大于 1，应弹出确认：

```text
确认删除 5 个标注框吗？此操作可撤销。
```

### 6.5 bbox 显示

bbox 默认显示：

```text
类别名 #TrackID
```

示例：

```text
person #12
```

显示样式：

- 普通框：类别颜色。
- 选中框：加粗边框。
- 插值框：虚线边框。
- 自动跟踪框：点划线边框或特殊标记。
- 异常框：红色警告标记。

### 6.6 复制和粘贴

支持：

- Ctrl + C：复制选中 bbox。
- Ctrl + V：粘贴到当前帧。
- Ctrl + Shift + V：复制上一帧所有 bbox 到当前帧。

粘贴时保留：

- 类别。
- Track ID。
- visibility。
- occlusion。
- ignore。
- note。

### 6.7 批量编辑

选中多个 bbox 后，右侧属性面板进入批量编辑模式。

支持批量修改：

- 类别。
- visibility。
- occlusion。
- ignore。
- note。

批量修改 Track ID 时应提示风险。

## 7. Track ID 管理需求

### 7.1 ID 自动分配

系统应提供“下一个可用 ID”功能。

默认规则：

- 新建 bbox 时自动分配全局递增 ID。
- 用户可以手动输入 ID。
- 从上一帧复制标注时保留原 Track ID。

### 7.2 ID 修改

用户修改 ID 时，应提供作用范围：

- 仅当前帧。
- 当前帧之后。
- 整个 Track。

默认选择“仅当前帧”，防止误操作。

### 7.3 ID 查找

支持通过输入框或快捷入口查找 Track ID。

查找结果显示：

- 首次出现帧。
- 最后出现帧。
- 总出现帧数。
- 类别。
- 是否存在异常。

用户点击结果后跳转到对应帧。

### 7.4 Track 生命周期

系统应为选中的 Track 显示基础生命周期信息：

- 起始帧。
- 结束帧。
- 出现次数。
- 插值帧数量。
- 自动跟踪帧数量。
- 异常帧数量。

## 8. 自动保存与恢复需求

### 8.1 自动保存默认开启

系统必须默认开启自动保存。

默认策略：

- 每 30 秒自动保存一次。
- 每次切换项目时自动保存。
- 每次导出前自动保存。
- 每次运行质量检查前自动保存。
- 程序关闭前自动保存。

### 8.2 自动保存文件

自动保存文件：

```text
annotations/internal.autosave.json
```

正式保存文件：

```text
annotations/internal.json
```

### 8.3 保存状态提示

界面必须清晰显示：

- 已保存。
- 有未保存修改。
- 正在自动保存。
- 自动保存成功。
- 自动保存失败。

自动保存失败时，应弹出非阻塞提示，并在状态栏显示错误。

### 8.4 崩溃恢复

软件启动或打开项目时，如果发现 autosave 文件比正式保存文件更新，应提示：

```text
检测到上次未正常关闭时的自动保存数据，是否恢复？

[恢复自动保存] [使用正式保存] [另存自动保存副本]
```

### 8.5 版本备份

系统应保留最近 10 个备份文件。

备份路径：

```text
annotations/backups/internal_YYYYMMDD_HHMMSS.json
```

### 8.6 手动保存

Ctrl + S 执行手动保存。

保存成功后：

- 更新 `internal.json`。
- 清除未保存状态。
- 状态栏显示最近保存时间。

## 9. 快捷键需求

### 9.1 快捷键设计原则

第一阶段只保留最基础、最高频的快捷键。

快捷键应尽量对齐常用软件：

- 保存、撤销、重做、复制、粘贴使用通用习惯。
- 播放暂停使用 Space。
- 帧切换使用 A/D 或方向键。
- 删除使用 Delete / Backspace。
- 导出使用 Ctrl + E。

### 9.2 默认快捷键表

| 功能 | 快捷键 |
|---|---|
| 新建项目 | Ctrl + N |
| 打开项目 | Ctrl + O |
| 保存 | Ctrl + S |
| 撤销 | Ctrl + Z |
| 重做 | Ctrl + Y |
| 复制 | Ctrl + C |
| 粘贴 | Ctrl + V |
| 删除选中框 | Delete / Backspace |
| 取消选择 | Esc |
| 播放 / 暂停 | Space |
| 上一帧 | A / ← |
| 下一帧 | D / → |
| 跳转到指定帧 | Ctrl + G |
| 放大 | Ctrl + + |
| 缩小 | Ctrl + - |
| 适应窗口 | Ctrl + 0 |
| 复制上一帧所有框 | Ctrl + Shift + V |
| 自动跟踪当前目标 | T |
| 运行质量检查 | F7 |
| 导出 | Ctrl + E |

### 9.3 类别快捷键

数字键 1 到 9 默认对应前 9 个类别。

按数字键时：

- 如果选中了 bbox，则修改选中 bbox 的类别。
- 如果未选中 bbox，则设置当前默认类别。

### 9.4 快捷键扩展

第一阶段可不实现复杂的快捷键自定义界面，但应将快捷键集中定义在单独配置或服务类中，便于后续扩展。

后续版本可支持：

- 查看所有快捷键。
- 修改快捷键。
- 检查快捷键冲突。
- 恢复默认快捷键。
- 导入 / 导出快捷键配置。

## 10. 自动跟踪辅助需求

### 10.1 功能定位

本项目需要包含类似自动跟踪的功能，但第一阶段不要求引入深度学习模型。

自动跟踪功能定位为：

> 用户在某一帧手动标注目标后，系统基于传统视觉跟踪算法或简单运动估计，在后续若干帧自动生成该目标的 bbox，用户再进行人工确认和修正。

### 10.2 自动跟踪模式

系统应支持至少两种自动跟踪模式。

#### 模式 1：单目标自动跟踪

用户选中当前帧的一个 bbox，点击“自动跟踪”或按 T。

系统弹出跟踪设置：

- 跟踪帧数，默认 30 帧。
- 跟踪方向：向后 / 向前。
- 跟踪算法：OpenCV CSRT / KCF / MOSSE，默认 CSRT。
- 是否遇到低置信度自动停止。
- 是否覆盖已有同 ID 标注。

点击开始后，系统逐帧生成同一 Track ID 的 bbox。

#### 模式 2：当前帧所有目标自动跟踪

用户点击“跟踪当前帧全部目标”。

系统对当前帧所有 bbox 分别启动跟踪器，并在后续帧生成对应 bbox。

默认最多跟踪 30 帧，防止误生成过多数据。

### 10.3 跟踪结果状态

自动跟踪生成的 bbox 必须标记为：

```json
"source": "tracker"
```

同时包含：

```json
"confirmed": false
```

未确认的自动跟踪框在界面中应使用特殊样式显示。

用户确认后：

```json
"confirmed": true
```

### 10.4 跟踪结果确认

系统应支持：

- 确认当前自动跟踪框。
- 确认当前帧所有自动跟踪框。
- 删除当前自动跟踪框。
- 删除当前 Track 的未确认跟踪结果。

### 10.5 跟踪过程显示

跟踪过程中应显示进度条：

```text
正在自动跟踪：第 12 / 30 帧
```

用户可以随时取消。

取消后已生成结果保留，但标记为未确认。

### 10.6 跟踪冲突处理

如果目标帧中已经存在相同 Track ID 的 bbox，系统应根据用户设置处理：

- 跳过已有标注。
- 覆盖已有标注。
- 保留已有标注并停止该 ID 跟踪。

默认策略：跳过已有标注。

### 10.7 跟踪失败处理

如果跟踪器失败或 bbox 越界严重，应停止该目标跟踪，并提示：

```text
Track #12 在第 135 帧跟踪失败，已停止生成后续结果。
```

失败条件包括：

- OpenCV tracker 返回失败。
- bbox 面积过小。
- bbox 超出图像范围比例过大。
- 连续帧 bbox 跳变超过阈值。

### 10.8 推荐实现方案

第一阶段可使用 OpenCV 内置 tracker：

- CSRT：精度较好，速度较慢，默认推荐。
- KCF：速度较快，适合一般场景。
- MOSSE：速度最快，精度较弱。

建议封装统一接口：

```python
class TrackerService:
    def track_single(self, start_frame_index, bbox, track_id, class_id, num_frames, direction, algorithm):
        pass

    def track_multiple(self, start_frame_index, annotations, num_frames, direction, algorithm):
        pass
```

## 15. 质量检查需求

### 15.1 基础质量检查

必须检查：

- bbox 坐标越界。
- bbox 宽度或高度小于等于 0。
- bbox 面积过小。
- 类别为空。
- Track ID 为空。
- 同一帧同一 Track ID 出现多次。
- 同一 Track ID 跨类别使用。
- 标注帧号超出范围。
- 未确认自动跟踪框。
- 未确认插值框。

### 15.2 轨迹质量检查

建议检查：

- Track 生命周期异常中断。
- 相邻帧中心点跳变过大。
- 相邻帧 bbox 面积变化过大。
- Track 长度过短。
- Track 中存在大量未确认框。

### 15.3 检查结果展示

质量检查结果以表格显示。

每条结果包含：

- 严重程度：错误 / 警告 / 提示。
- 问题类型。
- 帧号。
- Track ID。
- 类别。
- 描述。
- 建议操作。

点击结果后跳转到对应帧。

### 15.4 一键修复

可自动修复的问题：

- bbox 越界裁剪。
- 删除面积为 0 的 bbox。
- 删除空标注。

涉及 ID 修改的问题只给建议，不自动修复。

### 17.5 OpenCV Tracker 兼容处理

不同 OpenCV 版本 tracker 创建方法不同，应封装兼容函数：

```python
def create_tracker(name: str):
    name = name.upper()
    if name == "CSRT":
        if hasattr(cv2, "TrackerCSRT_create"):
            return cv2.TrackerCSRT_create()
        return cv2.legacy.TrackerCSRT_create()
    if name == "KCF":
        if hasattr(cv2, "TrackerKCF_create"):
            return cv2.TrackerKCF_create()
        return cv2.legacy.TrackerKCF_create()
    if name == "MOSSE":
        return cv2.legacy.TrackerMOSSE_create()
    raise ValueError(f"Unsupported tracker: {name}")
```

## 23. 总结

本项目应开发成一款面向多类别 MOT 数据生产的桌面端标注工具。相比传统 DarkLabel 类工具，本工具更强调傻瓜式操作、现代友好界面、自动保存、基础且常用的快捷键、Track ID 管理、质量检查和基于 OpenCV 的自动跟踪辅助。

该文档可直接作为 Codex 的开发需求输入。开发时应优先保证最小闭环可运行，再逐步实现自动保存、快捷键、插值、自动跟踪和质量检查。
